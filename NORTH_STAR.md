# .NeoN North Star

.NeoN is not a clone of an existing category.

It is a root system for human-origin intelligence.

## Core Claim

Human-created intelligence can become durable infrastructure.

When that happens, origin matters.
Lineage matters.
Context matters.
Proof matters.
Memory matters.

## What .NeoN Is Reaching Toward

.NeoN is a concentrated protocol seed that can unfold into:

- creator identity
- local memory
- artifact lineage
- proof packets
- workflow capture
- operational maps
- impact records
- machine-readable attribution
- human-readable origin stories
- future participation rails

## The Difference

Most tools manage work.

.NeoN preserves the origin of intelligence before work becomes invisible infrastructure.

Most systems track files, tasks, users, or money.

.NeoN tracks the living chain from human idea to deployed system.

## Design Law

Every .NeoN layer must answer at least one of these questions:

1. Who originated this?
2. What changed?
3. What did it become?
4. Where did it travel?
5. What value did it create?
6. What proof survives?
7. What human should remain attached to it?

## Not The Ceiling

Git is only the nearest historical analogy.
It is not the destination.

.NeoN is for intelligence artifacts that may begin as notes, prompts, workflows, code, browser actions, diagrams, conversations, datasets, or systems.

## First North Star

Attribution survives automation.

## Second North Star

Human-origin intelligence becomes portable capital.

## Third North Star

The creator carries the lineage, not the platform.
