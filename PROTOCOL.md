# .NeoN Protocol Draft

.NeoN is a provenance protocol for human-created operational intelligence.

The protocol defines structures for:

- artifacts
- commits
- lineage
- impact
- attribution

## Artifact

An artifact is a reusable unit of intelligence.

Examples:

- workflow
- prompt chain
- automation
- agent system
- process map
- orchestration graph

## Commit

A commit represents a signed contribution or modification.

## Ledger

A ledger is an append-only history of provenance events.

## Impact

Impact records measurable operational value.

Examples:

- hours saved
- execution count
- deployment count
- efficiency gain

## Design Principles

- local-first
- append-only provenance
- portable identity
- human-readable structures
- machine-readable schemas
