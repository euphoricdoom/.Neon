# Golden Reference Objects

Golden objects are stable examples for the `.NeoN` protocol.

They exist to verify:

- artifact structure
- deterministic hashing
- lineage behavior
- proof packet shape
- CAS addressing
- graph output

## Rule

A protocol without stable reference objects eventually drifts.

Golden objects act as continuity anchors.

## v0.1 Targets

- root artifact
- derived artifact
- proof packet
- lineage graph
- CAS object
