# Contributing to .NeoN

.NeoN is a protocol project for persistent human-origin intelligence lineage.

## What Belongs Here

Contributions should strengthen at least one core property:

- origin
- lineage
- proof
- portability
- memory
- attribution
- creator continuity

## Contribution Types

You can contribute:

- protocol notes
- schemas
- examples
- CLI code
- vault design
- browser capture ideas
- proof packet formats
- trust model improvements
- documentation
- tests

## Design Rules

1. Local-first by default.
2. Agnostic by design.
3. Human-readable before complex.
4. Machine-readable when possible.
5. No private user data in public artifacts.
6. No vendor lock-in.
7. Do not invent cryptography when proven tools exist.
8. Prefer small composable primitives.

## Pull Request Expectations

Every pull request should include:

- clear purpose
- files changed
- why it strengthens .NeoN
- any privacy or trust implications
- tests or validation notes when applicable

## Language

Use precise language. Avoid hype when writing specs.

The project can be bold without being vague.
