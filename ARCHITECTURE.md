# .NeoN Architecture Overview

## Core Principle

Attribution survives automation.

## Architectural Shape

.NeoN is built as a layered protocol ecosystem.

## Layer 1 — Primitive Objects

Core portable structures:
- artifacts
- commits
- ledgers
- proof packets
- identities
- impact records

## Layer 2 — Local Sovereign Vault

Creator-owned local storage.

Potential technologies:
- SQLite
- local files
- append-only logs

## Layer 3 — Capture Systems

Systems that record human-origin intelligence.

Examples:
- browser extension
- CLI capture
- workflow import
- AI prompt capture

## Layer 4 — Lineage Engine

Maps:
- derivation
- reuse
- relationships
- influence
- ancestry

## Layer 5 — Proof and Trust

Systems for:
- signatures
- hashes
- verification
- timestamp anchoring
- confidence scoring

## Layer 6 — Ecosystem Interfaces

Possible integrations:
- GitHub
- local AI systems
- workflow tools
- enterprise systems
- APIs
- SDKs

## Design Constraints

- local-first
- agnostic by design
- portable artifacts
- minimal trust assumptions
- redaction-friendly
- composable primitives

## Important Distinction

.NeoN does not attempt to own all infrastructure.

It creates a lineage and provenance layer that can survive across infrastructure changes.
