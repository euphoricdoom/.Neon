# .NeoN Governance

.NeoN is intended to become an open protocol ecosystem.

## Current Stage

The project is currently founder-led.

Founder:
- Carl Sowers

## Long-Term Goal

Move toward a transparent protocol governance model that protects:

- creator sovereignty
- interoperability
- portability
- protocol neutrality
- open standards

## Governance Principles

1. No single vendor should control the protocol.
2. Public specifications should remain accessible.
3. Local-first creator ownership is a core value.
4. Privacy and provenance must coexist.
5. The protocol should remain implementation-agnostic.
6. Contributors should receive attribution.

## Protocol Stability

Changes to foundational primitives should be conservative.

Examples:
- artifact structure
- proof packet format
- lineage semantics
- identity mapping rules

## Future Direction

Potential future governance structures:
- working groups
- specification councils
- public RFC process
- ecosystem interoperability tests
