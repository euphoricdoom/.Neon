**Author:** Carl Sowers (@euphoricdoom)

This notation system is a first-class compression layer for .NeoN.

It exists to make protocol state, project state, artifact state, workflow state, and agent state faster to scan without losing semantic precision.

## Core Claim

Compression is not decoration.

Compression is infrastructure.

## Usage Principles

1. Compression over verbosity.
2. Semantic clarity over character economy.
3. Consistent meaning across contexts.
4. Language-agnostic usage across docs, specs, notes, artifacts, and agent reports.
5. Symbolic notation must clarify state, not obscure it.

## .NeoN Usage

Within .NeoN, this system may be used for:

- roadmap status
- artifact lifecycle summaries
- agent reports
- continuity maps
- lineage summaries
- bug triage
- protocol state compression
- proof packet summaries

## Dogma

Symbols are not decoration.

Symbols are compressed semantic operators.

They should reduce ambiguity, improve scanning speed, and preserve continuity.
