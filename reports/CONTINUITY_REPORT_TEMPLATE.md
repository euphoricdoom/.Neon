# .NeoN Continuity Report Template

A continuity report compresses current project state without breaking semantic clarity.

## Symbolic State

```text
⚡ π.<current-phase>
● ∫ <active-build>
◐ λ.<partial-system>
✓ λ.<solved-system>
✗ θ.<blocked-risk>
† ƒ <deprecated-file>
```

## Human State

### Current Phase

Describe the active phase in plain language.

### Active Work

List what is being built now.

### Solved

List what is working and should not be re-litigated.

### Partial

List what exists but still needs hardening.

### Blocked

List hard blockers and why they matter.

### Deprecated

List old files, patterns, or assumptions that should not guide future work.

## Continuity Check

Answer yes or no:

- Does origin survive?
- Does derivation preserve ancestry?
- Does hashing remain deterministic?
- Does CAS preserve bytes?
- Does proof export verify?
- Does graph output reveal lineage?
- Does documentation match implementation?

## Next Manipulation

Name the next smallest complexity to manipulate.

Examples:

- malformed proof packet
- broken parent reference
- symbolic report generation
- CAS URI verification
- README compression
- signed release workflow
