# .NeoN Primitive Fission Doctrine

.NeoN starts with compressed primitives.

Primitives split only when pressure proves the split is necessary.

## Core Rule

Only split primitives that remain compatible and complete each other.

A split is valid only if each new primitive:

1. preserves the original lineage intent
2. strengthens clarity
3. improves verification
4. remains portable
5. composes with the others
6. does not create platform lock-in
7. does not erase origin

## Bad Split

A bad split creates isolated parts that do not recombine cleanly.

Signs of a bad split:

- duplicated identity fields
- conflicting hashes
- unclear ownership of lineage
- unverifiable proof boundaries
- hidden vendor assumptions
- incompatible formats

## Good Split

A good split creates complementary primitives.

Example:

```text
artifact.neon  = what exists
lineage.neon   = where it came from
proof.neon     = why it can be trusted
identity.neon  = who remains attached
impact.neon    = what value it created
```

Each file can stand alone, but together they form a stronger object.

## Fission Law

Compression first.
Fission under pressure.
Compatibility always.

## Design Test

Before splitting a primitive, ask:

1. Does the split reduce ambiguity?
2. Does it preserve lineage?
3. Does it improve proof?
4. Can the parts recombine without loss?
5. Can a human still understand it?

If not, do not split.
